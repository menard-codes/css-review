Viewport Meta

Gives the browser an "instruction" on how to control the page's dimension and scaling

<meta name='viewport' content='width=device-width, initial-scale=1.0' />

The width=device-width sets the width of the page to fit the screen-width

The initial-scale=1.0 sets the initial zoom level when the page is first loaded by the browser

In addition to initial-scale, you can also set the following attributes:

  • minimum-scale
  • maximum-scale
  • user-scalable

These can disable the user's ability to zoom the viewport, potentially causing accessibility issues, therefore they're not recommended

Size Content to Viewport

  1. Do NOT use large fixed width elements - If the element is wider than the viewport, it may cause horizontal scroll
  2. Do NOT let the content rely on a particular viewport width to render well - Screen dimensions vary, so it shall not depend on a particular viewport to render.
  3. Use CSS media queries to apply different styling for small and large screens - Setting large absoulte CSS widths for page elements will cause it to be too wide on smaller devices Instead, consider using relative values (i.e. width: 100%). Be careful of using large absolute positioning values as it may cause the element to fall outside the viewport on smaller devices

Images

Images should be set to max-width so that it won't stretch larger than its natural size

img { max-width: 100%; display: block }

Layout

Screen dimensions vary, content should not rely on a particular viewport width to render. for using percentages in the widths, the columns always remain a certain percentage of the container
