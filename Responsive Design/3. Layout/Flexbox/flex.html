Flexbox Vendor Prefexing

Flexbox changed overtime, so different browsers support different versions of flexbox. That's why it's necessary to add vendor prefixing

Flex Properties

Container props:

flex-wrap

It defines whether the flex items are forced in a single line or can be flowed into multiple lines. If set to multiple lines, it also defines the cross-axis which determines the direction new lines are stacked in. nowrap - all flex items will be on one line wrap - flex items will wrap onto multiple lines from top to bottom wrap-reverse - flex items wrap into multiple lines from bottom to top

No Wrap

Wrap

Wrap Reverse

Flex Flow

Short-hand for flex-direction and flex wrap. Default value is: row nowrap

Align Items vs Align Content

Align Items - aligns the items themselves in a row.

Now, align-content doesn't interfere with items but in the row themselves. Hence, align-content will try to align rows with respect to each other and flex container.

Children Props

order

by default, flex items are ordered based on the source, but you can reorder them on CSS using order property

flex-grow

This is similar to approach of css grid to fill in the extra spaces. The default is 1, where space is equally distributed, but if one item has a value of 2 for example, it will occupy twice as more

flex-shrink

Inverse of flex-grow

flex

Shorthand for flex-grow, flex-shrink, flex-basis

align-self

Overwrites the value set by the parent/container. Value provided will be used to align this item
